# 来源质量标准

> 本文是人读质量说明；进入链路和质量门由 `system/knowledge-ingestion-system.json` 统筹，外部知识审查由 review-system 接管。

## 可信度层级

| 层级 | 类型 | 使用方式 |
| --- | --- | --- |
| A | 系统综述、权威标准、官方文档、高质量基准报告 | 可支撑强判断 |
| B | 同行评议论文、成熟开源项目、主流技术文档 | 可支撑一般判断 |
| C | 技术博客、会议演讲、个人经验、issue 讨论 | 可作为线索或实践补充 |
| D | 未验证传闻、营销材料、匿名观点 | 只能作为待验证假设 |

## 记录原则

- `confidence` 表示当前知识条目的可信程度，不表示来源本身永远可靠。
- 对变化快的内容使用 `last_checked_at`，例如 GitHub、价格、模型能力、API 限制。
- 对争议内容写清反例和适用边界。
- 对论文写清样本、方法和局限。
- 对工程实践写清技术栈、版本和运行环境。

## 引用方式

在知识条目中优先引用 `source record id`，不要只贴 URL。这样未来 URL 失效时，仍能通过来源记录找回上下文。

示例：

```yaml
sources:
  - paper-20260605-example-title
  - github-20260605-example-repo
```

正文中可以写：

```markdown
这个判断主要来自 `paper-20260605-example-title`，并由 `github-20260605-example-repo` 的工程实践信号补充。
```
