# 知识获取与入库决策

本文是人读说明。机器真相由 `system/knowledge-acquisition-system.json` 接管，候选动态数据由 `10-sources/acquisition-queue.json` 接管。

## 作用

这个子系统负责知识进入前的第一道判断：系统不只等待用户丢进来源，也可以根据项目需求、领域缺口、审查发现或诊断结果主动寻找候选知识，并判断它是否值得进入知识库。

## 基本流程

1. 触发：用户材料、项目需求、领域空白、审查发现或诊断缺口。
2. 寻找：优先找权威教材、综述、官方文档、标准、优秀 GitHub 项目和高质量论文。
3. 初评：评价来源权威性、证据强度、基础价值、复用价值、时效、领域适配、风险和获取成本。
4. 决策：登记、先审查、继续寻找、观望或拒绝。
5. 交接：通过后进入来源登记、领域路由、审查系统、loop 或父级知识摄入链路。

## 边界

- 候选队列不是可引用知识事实。
- 正式引用必须回到 source record、atomic note、synthesis 或 catalog。
- 高风险、高不确定、高价值但证据不足的候选先交给 review-system。
- 重复性的权威来源寻找任务可以交给 loop-orchestration。
